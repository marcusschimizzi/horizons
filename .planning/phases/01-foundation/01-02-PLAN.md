Create `src/db/schema.ts` with the tasks table. **CRITICAL: `horizon` is NOT a column — it is computed client-side from `targetDate` + `now`.**

```typescript
import { pgTable, text, timestamp, boolean, integer } from 'drizzle-orm/pg-core';
import { createId } from '@paralleldrive/cuid2';

export const tasks = pgTable('tasks', {
  id: text('id').primaryKey().$defaultFn(() => createId()),
  rawInput: text('raw_input').notNull(),
  title: text('title').notNull(),
  // horizon is NOT stored — computed client-side from targetDate + now
  targetDateEarliest: timestamp('target_date_earliest', { mode: 'date', withTimezone: true }),
  targetDateLatest: timestamp('target_date_latest', { mode: 'date', withTimezone: true }),
  hardDeadline: timestamp('hard_deadline', { mode: 'date', withTimezone: true }),
  needsRefinement: boolean('needs_refinement').notNull().default(false),
  refinementPrompt: text('refinement_prompt'),
  status: text('status', { enum: ['active', 'completed', 'dropped'] }).notNull().default('active'),
  createdAt: timestamp('created_at', { mode: 'date', withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp('updated_at', { mode: 'date', withTimezone: true }).notNull().defaultNow().$onUpdateFn(() => new Date()),
  driftCount: integer('drift_count').notNull().default(0),
  tags: text('tags').array(),
});
```

Create `src/db/index.ts` with the globalThis singleton pattern to prevent HMR connection pool exhaustion:

```typescript
import { drizzle } from 'drizzle-orm/node-postgres';
import { Pool } from 'pg';
import * as schema from './schema';

const globalForDb = globalThis as unknown as { pool: Pool | undefined };

const pool = globalForDb.pool ?? new Pool({
  connectionString: process.env.DATABASE_URL,
  max: 10,
  ssl: process.env.NODE_ENV === 'production' ? { rejectUnauthorized: false } : false,
});

if (process.env.NODE_ENV !== 'production') {
  globalForDb.pool = pool;
}

export const db = drizzle(pool, { schema });
export type Database = typeof db;
```

**Why globalThis?** During development, Next.js hot-reloads modules. Without the singleton, every reload creates a new Pool, exhausting Postgres connections. The globalThis pattern reuses the same Pool across HMR reloads.

**Why SSL only in production?** Local Docker Compose Postgres doesn't use SSL. Railway Postgres requires SSL with `rejectUnauthorized: false` because it uses self-signed certificates.

Ensure Docker Compose Postgres is running (`docker compose up -d`), then run `npm run db:push` to apply the schema.
