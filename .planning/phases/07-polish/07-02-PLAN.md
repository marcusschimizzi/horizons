Modify TaskSprite.tsx to add three visual indicators:

**1. Drift opacity degradation:**
In the existing useFrame callback, after the dissolution and entrance animation logic, add drift-based opacity reduction for steady-state rendering. When the sprite is not animating (not dissolving, not new), set:
```typescript
// Apply drift-based opacity reduction in steady state
if (!dissolvingRef.current && !isNew && materialRef.current) {
  const driftReduction = Math.min(task.driftCount * 0.08, 0.5);
  const baseOpacity = SCENE_CONSTANTS.spriteOpacity;
  materialRef.current.opacity = Math.max(0.4, baseOpacity - driftReduction);
}
```
This reduces opacity by 0.08 per drift, from 0.9 base down to a floor of 0.4.

Also desaturate the glowColor more as drift increases. Modify the existing `glowColor` memo to factor in drift:
```typescript
const glowColor = useMemo(() => {
  const firstTag = task.tags?.[0];
  const baseHex = firstTag && isTagCategory(firstTag) ? TAG_COLORS[firstTag] : DEFAULT_COLOR;
  const color = new THREE.Color(baseHex);
  // More drift = more desaturation toward ethereal target
  const driftLerp = Math.min(0.25 + (task.driftCount ?? 0) * 0.1, 0.7);
  color.lerp(ETHEREAL_TARGET, driftLerp);
  color.multiplyScalar(SCENE_CONSTANTS.spriteEmissiveMultiplier);
  return color;
}, [task.tags, task.driftCount]);
```
