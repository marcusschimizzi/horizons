**TaskNode.tsx:**

Create `src/components/TaskNode.tsx` — the LOD controller that decides whether a task renders as a card or a sprite.

Props:
```ts
interface TaskNodeProps {
  task: Task;
  position: [number, number, number];
}
```

Implementation:
- Import `SCENE_CONSTANTS` from scene-constants.ts.
- Create a `Set` from `SCENE_CONSTANTS.cardHorizons` for O(1) lookup.
- Check if `task.horizon` is in the card horizons set.
- If yes: render `<TaskCard task={task} position={position} />`
- If no: render `<TaskSprite task={task} position={position} />`

This is a categorical split, NOT distance-based. No camera distance calculation needed. The horizon is already computed on the task object.

**Crossfade / anti-pop (SCENE-05):**
In Phase 3, the LOD split is purely categorical — it does not change dynamically because the camera cannot move yet (Phase 4). Therefore, crossfade animation is NOT needed in this plan. However, design the component so that Phase 4 can add camera-distance-based LOD switching with crossfade later. Specifically:
- Keep the card/sprite decision in a variable (`const isCard = ...`) rather than inlining it.
- Phase 4 will extend this to include camera distance + hysteresis buffer.

**DebugOverlay.tsx:**

Create `src/components/DebugOverlay.tsx` — a plain HTML overlay that displays current scene constants.

Implementation:
- Check URL for `?debug=true` using `useEffect` + `useState` (read `window.location.search` on mount).
- If not debug mode, render `null`.
- If debug mode, render a fixed-position div (top-right corner, z-index 9999) with:
  - Monospace font, small text (11px), dark semi-transparent background
  - Display key constants from SCENE_CONSTANTS: fogDensity, bloomIntensity, bloomLuminanceThreshold, spriteBaseRadius, spriteEmissiveMultiplier, htmlDistanceFactor
  - Show task count breakdown: total tasks, cards, sprites
- Accept a `taskBreakdown` prop: `{ total: number; cards: number; sprites: number }`.
- This is a plain DOM component, NOT inside the Canvas. It renders as a sibling to the Canvas in HorizonScene.

**Do NOT:**
- Add camera distance calculation to TaskNode (Phase 4)
- Add crossfade animation (not needed until camera moves in Phase 4)
- Make DebugOverlay interactive (sliders come later if needed)
- Place DebugOverlay inside the R3F Canvas
