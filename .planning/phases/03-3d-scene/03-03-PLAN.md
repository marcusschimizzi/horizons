Create `src/components/TaskCard.tsx` that renders a single near-horizon task as an HTML card in 3D space.

**Props:**
```ts
interface TaskCardProps {
  task: Task;
  position: [number, number, number];
}
```

**Implementation:**

1. **Outer structure:**
   - `<group position={position}>` wrapping the entire component.
   - `<Html center distanceFactor={SCENE_CONSTANTS.htmlDistanceFactor} style={{ pointerEvents: 'none' }}>` for the CSS overlay.
   - `pointerEvents: 'none'` is critical — cards are display-only in Phase 3. Phase 6 will change this.

2. **Card styling (inline styles, not Tailwind — drei Html renders outside Tailwind's scope):**
   - Width: ~200px
   - Padding: 10px 14px
   - Background: `rgba(18, 18, 26, 0.75)` — frosted glass on dark
   - backdropFilter: `blur(8px)` — frosted effect
   - Border: `1px solid rgba(148, 163, 184, 0.15)` — subtle cool-gray border
   - borderRadius: 8px
   - Color: `#e2e8f0` — light text
   - fontSize: 13px, lineHeight: 1.3
   - whiteSpace: nowrap, overflow: hidden, textOverflow: ellipsis — single-line title, truncated

3. **Hard-deadline ring (VIS-03):**
   - If `task.hardDeadline !== null`, add a boxShadow: `0 0 8px rgba(245, 158, 11, 0.4), inset 0 0 4px rgba(245, 158, 11, 0.1)` — steady amber glow, not pulsing.
   - This is the one warm color in the otherwise cool palette. It says "this one matters."

4. **Drift indicator:**
   - If `task.driftCount > 0`, apply CSS filter: `saturate(${Math.max(0.3, 1 - task.driftCount * 0.15)})` — desaturates progressively.
   - Also reduce opacity: `Math.max(0.6, 1 - task.driftCount * 0.1)` — subtly dims.
   - The effect is atmospheric: drifted tasks look slightly worn, faded, like they've been sitting too long. Not accusatory, just... tired.

5. **Content:** Just `{task.title}` as text content. Nothing else. No tags, no dates, no badges. Title only per CONTEXT.md decision.

**Do NOT:**
- Use Tailwind classes — drei Html renders outside the normal DOM tree and Tailwind classes may not apply reliably. Use inline styles.
- Add onClick or any interaction handlers — Phase 6.
- Show tags, date range, or drift count as text — title only per design decision.
- Use `<Billboard>` — Html handles its own DOM positioning. Wrapping in Billboard is unnecessary and hurts performance.
- Use `transform` prop on Html — not needed for this use case and can cause z-fighting.
