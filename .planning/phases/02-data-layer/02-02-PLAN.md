  <name>Task 1: Create Zustand vanilla store with provider, hooks, and derived selectors</name>
  <files>
    src/stores/task-store.ts

  <action>
    Create `src/stores/task-store.ts` with `'use client'` directive at the top.

    **Use the vanilla createStore pattern (NOT `create` from zustand)** — this is critical for Next.js App Router to prevent SSR state leaks between requests.

    **Store types:**
    ```
    TaskState: { tasks: TaskRow[] }
    TaskActions: { setTasks, addTask, updateTask, removeTask, refresh }
    TaskStore = TaskState & TaskActions
    ```

    **Store factory function `createTaskStore(initialTasks: TaskRow[])`:**
    - Uses `createStore` from `zustand/vanilla` (NOT `create` from `zustand`).
    - Initializes `tasks` with `initialTasks` parameter.
    - Actions:
      - `setTasks(tasks: TaskRow[])` — replace all tasks.
      - `addTask(task: TaskRow)` — append a single task.
      - `updateTask(id: string, updates: Partial<TaskRow>)` — update a task by ID in the array.
      - `removeTask(id: string)` — remove a task by ID from the array.
      - `refresh()` — `async` action that fetches `GET /api/tasks`, deserializes dates, and calls `set({ tasks })`. Must reconstruct Date objects from ISO strings using a `deserializeTask` helper.

    **Date deserialization helper (define inside the file):**
    ```typescript
    function deserializeTask(raw: Record<string, unknown>): TaskRow {
      return {
        ...raw,
        targetDateEarliest: raw.targetDateEarliest ? new Date(raw.targetDateEarliest as string) : null,
        targetDateLatest: raw.targetDateLatest ? new Date(raw.targetDateLatest as string) : null,
        hardDeadline: raw.hardDeadline ? new Date(raw.hardDeadline as string) : null,
        createdAt: new Date(raw.createdAt as string),
        updatedAt: new Date(raw.updatedAt as string),
      } as TaskRow;
    }
    ```

    **Context + Provider:**
    - Create `TaskStoreContext` via `createContext<ReturnType<typeof createTaskStore> | null>(null)`.
    - Create `TaskStoreProvider` component accepting `{ children, initialTasks }`.
    - Use `useRef<ReturnType<typeof createTaskStore>>(undefined)` to hold the store instance. Initialize lazily: `if (!storeRef.current) { storeRef.current = createTaskStore(initialTasks); }`.
    - Render `<TaskStoreContext.Provider value={storeRef.current}>{children}</TaskStoreContext.Provider>`.

    **Base hook:**
    - `useTaskStore<T>(selector: (state: TaskStore) => T): T` — gets store from context, throws if null, returns `useStore(store, selector)`.
    - Import `useStore` from `zustand`.

    **Derived selectors (exported as named functions):**

    `useTasksWithHorizon(): Task[]`
    - Uses `useTaskStore` to select `state.tasks`.
    - Maps each `TaskRow` to `Task` by computing `horizon` via `getHorizon()` from `@/lib/horizons`.
    - Constructs `targetDate: DateRange | null` from `targetDateEarliest`/`targetDateLatest`.
    - Uses `new Date()` as `now` for horizon computation.

    `useTasksByHorizon(): Map<Horizon, Task[]>`
    - Calls `useTasksWithHorizon()`.
    - Groups into a `Map<Horizon, Task[]>` keyed by the 6 horizon values.

    **Exports:** `TaskStoreProvider`, `useTaskStore`, `useTasksWithHorizon`, `useTasksByHorizon`, `deserializeTask` (export for potential reuse in Phase 6).

    **Type imports:** Import `TaskRow`, `Task`, `DateRange` from `@/types/task`. Import `Horizon`, `getHorizon` from `@/lib/horizons`. Do NOT import anything from `@/db` — this is a client module.
  </action>

    ```typescript
    import { db } from '@/db';
    import { tasks } from '@/db/schema';
    import SceneLoader from '@/components/SceneLoader';

    export default async function Home() {
      try {
        const allTasks = await db.select().from(tasks);
        return <SceneLoader initialTasks={allTasks} />;

        console.error('Failed to fetch tasks:', e);
        return <SceneLoader initialTasks={[]} error />;
      }
    }
    ```
