  <action>
    1. Install the `server-only` package: `npm install server-only`. Then add `import 'server-only';` as the FIRST import in `src/db/index.ts` to prevent accidental client-side import of the database module.

    2. Create `src/app/api/tasks/route.ts` with two exports:

       **GET handler:**
       - Select all rows from the tasks table (`db.select().from(tasks)`) — no status filter (return all tasks including completed/dropped for now).
       - Wrap in try/catch. Return `Response.json(allTasks)` on success.
       - On error: return `{ error: 'Failed to fetch tasks', code: 'INTERNAL_ERROR' }` with status 500.

       **POST handler:**
       - Parse request body via `await request.json()`.
       - Validate that `title` and `rawInput` are present and are non-empty strings. If missing, return `{ error: 'title and rawInput are required', code: 'VALIDATION_ERROR' }` with status 400.
       - Insert into tasks table using `db.insert(tasks).values(body).returning()`. Destructure to get `[created]`.
       - Return `Response.json(created, { status: 201 })`.
       - On error: return `{ error: 'Failed to create task', code: 'INTERNAL_ERROR' }` with status 500.

    3. Create `src/app/api/tasks/[id]/route.ts` with three exports:

       **CRITICAL: Next.js 16 params are a Promise.** The second argument signature MUST be:
       ```typescript
       { params }: { params: Promise<{ id: string }> }
       ```
       And you MUST `const { id } = await params;` inside each handler. Forgetting this causes `id` to be `undefined` silently.

       **GET handler:**
       - `const { id } = await params;`
       - `const [task] = await db.select().from(tasks).where(eq(tasks.id, id));`
       - If no task found: return `{ error: 'Task not found', code: 'TASK_NOT_FOUND' }` with status 404.
       - Return `Response.json(task)`.

       **PATCH handler:**
       - `const { id } = await params;`
       - Parse body. Build an update object containing only known task column keys from the body (strip unknown fields). Known updatable fields: `title`, `rawInput`, `targetDateEarliest`, `targetDateLatest`, `hardDeadline`, `needsRefinement`, `refinementPrompt`, `status`, `driftCount`, `tags`.
       - If the update object is empty after stripping, return `{ error: 'No valid fields to update', code: 'VALIDATION_ERROR' }` with status 400.
       - `const [updated] = await db.update(tasks).set(updateFields).where(eq(tasks.id, id)).returning();`
       - If no row returned: return 404 with `TASK_NOT_FOUND`.
       - Return `Response.json(updated)`.

       **DELETE handler:**
       - `const { id } = await params;`
       - `const [deleted] = await db.delete(tasks).where(eq(tasks.id, id)).returning();`
       - If no row returned: return 404 with `TASK_NOT_FOUND`.
       - Return `Response.json(deleted)`.

    4. All route handlers import from `@/db` and `@/db/schema`. Import `eq` from `drizzle-orm`. Use the `NextRequest` type from `next/server` or plain `Request` — either works, but be consistent.

    5. Error shape is always `{ error: string, code: string }` per CONTEXT.md.
  </action>

    Create `src/db/seed.ts` — the seed script run via `npm run db:seed` (already wired in package.json as `tsx src/db/seed.ts`).

    3. Define the seed data array inline.
    4. `async function seed()`: wipe with `await db.delete(tasks)`, then `await db.insert(tasks).values(seedData)`.
    5. Log the count and call `process.exit(0)`. Catch errors, log, exit 1.

    **Seed data requirements (~35 tasks):**

    Distribution by horizon (dates relative to "now" — use `new Date()` at script start and compute dates from it):
    - **Immediate** (~8 tasks): targetDate within 0-1 days. Examples: "Reply to Sarah's email", "Pick up prescription", "Call dentist to confirm", "Submit expense report", "Fix leaking kitchen faucet", "Buy birthday gift for Mom", "Review lease renewal letter", "Prep slides for Monday standup".
    - **This Week** (~7 tasks): targetDate 2-7 days out. Examples: "Dentist appointment Thursday", "Grocery run for the week", "Schedule car oil change", "Finish quarterly self-review", "Return library books", "Call landlord about AC filter", "Plan weekend hike route".
    - **This Month** (~6 tasks): targetDate 8-30 days out. Examples: "File Q1 taxes", "Book flights for spring trip", "Annual physical checkup", "Replace bathroom caulking", "Set up automated savings transfer", "Coffee catch-up with Jake".
    - **This Quarter** (~6 tasks): targetDate 31-90 days out. Examples: "Plan summer vacation itinerary", "Get quotes for fence repair", "Renew passport", "Research 401k rebalancing", "Deep clean garage", "Sign up for pottery class".
    - **This Year** (~5 tasks): targetDate 91-365 days out. Examples: "Train for fall half-marathon", "Learn basic conversational Spanish", "Build raised garden beds", "Replace living room carpet", "Plan Thanksgiving hosting".
    - **Someday** (~3-4 tasks): no targetDate (null for both earliest/latest). Examples: "Write a short story", "Visit Japan", "Learn to sail", "Build a custom desk".

    **Date ranges:** For tasks with dates, set `targetDateEarliest` and `targetDateLatest` with a realistic spread:
    - Immediate: same day or +1 day spread
    - This Week: 1-2 day spread
    - This Month: 2-5 day spread
    - This Quarter: 3-7 day spread
    - This Year: 5-14 day spread
    - Someday: both null

    **Special states (organic distribution):**
    - `hardDeadline`: Set on ~6-8 tasks. More likely on near-term tasks. Example: expense report deadline, tax filing, dentist appointment, lease renewal.
    - `driftCount > 0`: Set on ~5-7 tasks. Higher counts on farther tasks (they've been pushed more). Example: "Deep clean garage" driftCount: 3, "Renew passport" driftCount: 2.
    - `needsRefinement: true`: Set on ~3-4 tasks. Example: "Plan summer vacation itinerary", "Research 401k rebalancing", "Train for fall half-marathon".
    - `refinementPrompt`: Set on tasks with `needsRefinement: true`. Example: "What specific destinations are you considering? Break this into bookable steps."

    **Tags:** All six categories represented (work, personal, health, finance, home, social). Each task gets exactly one tag in the array. Distribution should feel natural — more personal/work, fewer social/finance.

    **All tasks have `status: 'active'`** (seed represents a clean working state).

    Use relative date computation so the seed data stays valid regardless of when it's run:
    ```typescript
    const now = new Date();
    const daysFromNow = (d: number) => {
      const date = new Date(now);
      date.setDate(date.getDate() + d);
      return date;
    };
    ```
