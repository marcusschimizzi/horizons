    2. Add camera constants to `src/lib/scene-constants.ts` (add to the existing SCENE_CONSTANTS object, do not replace existing fields):
       - `cameraRestZ: 10` — matches current Canvas camera prop `position: [0, 0, 10]`
       - `nearBoundary: 10` — Z=10 is the present; camera should not scroll past this toward the viewer
       - `farBoundary: -120` — end of 'someday' band per HORIZON_BANDS in horizons.ts
       - `maxOverscroll: 3` — max Z-units the rubber-band can stretch past nearBoundary
       - `zSmoothTime: 0.25` — smoothTime for damp3 on Z-axis movement (lower = snappier)
       - `zUnitsPerPixel: 0.08` — scroll delta multiplier (tune for 5-8 z-units per normal scroll tick)
       - `springBackSmoothTime: 0.15` — faster smoothTime for the spring-back from overscroll

    3. Create `src/stores/camera-store.ts` as a Zustand vanilla store (NOT hook-based `create`, use `createStore` from `zustand/vanilla`):
       - State: `targetZ: number`, `currentZ: number`, `velocity: number`, `isAnimating: boolean`
       - Initial values: targetZ and currentZ both set to `SCENE_CONSTANTS.cameraRestZ`
       - Actions:
         - `scroll(deltaZ: number)`: Adds deltaZ to targetZ. If newTargetZ > nearBoundary, apply rubber-band formula: `dampedOverscroll = maxOverscroll * (1 - Math.exp(-overscroll / maxOverscroll))` where overscroll = newTargetZ - nearBoundary, then newTargetZ = nearBoundary + dampedOverscroll. If newTargetZ < farBoundary, clamp to farBoundary. Set velocity = deltaZ, isAnimating = true.
         - `tick(newZ: number, settled: boolean)`: If settled, snap currentZ to targetZ, set isAnimating=false, velocity=0. Otherwise set currentZ = newZ.
       - Export `cameraStore` (the raw store) and `useCameraStore` hook (using `useStore` from zustand for React subscriptions).
       - This file must NOT have a .tsx extension (no JSX). Use .ts.
       - Do NOT import from or depend on task-store.tsx — these stores are independent.

  <action>
    1. Create `src/components/CameraRig.tsx`:
       - This is a null-returning R3F component rendered inside the Canvas.
       - Import `damp3` from `maath/easing`. Import `normalizeWheel` (default export) from `normalize-wheel-es`. Import `cameraStore` from camera-store.ts. Import scene constants.
       - **Wheel event handler** (via useEffect on `gl.domElement`):
         - Listen for `wheel` event with `{ passive: false }`. Call `e.preventDefault()` to prevent page scroll.
         - Use `normalizeWheel(e).pixelY` for cross-browser delta normalization.
         - Call `cameraStore.getState().scroll(-pixelY * SCENE_CONSTANTS.zUnitsPerPixel)`. Negative because scrolling down (positive pixelY) should move camera into the future (negative Z).
         - Call `invalidate()` to kick the render loop.
       - **useFrame loop**:
         - Read store state via `cameraStore.getState()` (non-reactive — no React re-renders).
         - If `!isAnimating`, return early (no work when camera is settled).
         - **Spring-back check**: If `targetZ > nearBoundary` AND `Math.abs(velocity) < 0.5` (user stopped scrolling past the boundary), set targetZ back to nearBoundary. This triggers the spring-back.
         - **Damping**: Use `damp3(camera.position, [0, 0, targetZ], smoothTime, delta)` to smoothly move the camera. Use `zSmoothTime` normally, but use `springBackSmoothTime` if camera is currently past nearBoundary and targetZ is nearBoundary (spring-back in progress).
         - **Note on damp3**: damp3 returns a boolean (true = still animating). It mutates camera.position in place. The target is `[0, 0, targetZ]` — X and Y are 0 for now (parallax will be added in Plan 02).
         - **Settle detection**: Check if `Math.abs(camera.position.z - targetZ) < 0.01`. If settled, call `cameraStore.getState().tick(targetZ, true)`. Otherwise call `tick(camera.position.z, false)` and call `invalidate()` to request next frame.
       - The component returns `null`.

    2. Wire CameraRig into `src/components/HorizonScene.tsx`:
       - Import CameraRig from `./CameraRig`.
       - Add `<CameraRig />` inside the `SceneContents` component, BEFORE `<TaskNodes />` (order does not matter for R3F but keeps it organized).
       - IMPORTANT: Remove the static `camera` prop from `<Canvas>`. Instead, set a `camera` prop with only `fov: 60` and `position: [0, 0, SCENE_CONSTANTS.cameraRestZ]`. This way the initial camera position uses the constant, and CameraRig takes over from there.
       - No other changes to HorizonScene.tsx.

    3. Verify the demand-mode invalidation loop is correct:

       - useFrame runs, damp3 moves camera, calls invalidate() if not settled
       - When settled, stops calling invalidate() -> render loop stops
       - SceneInvalidator (existing) handles task store changes separately — no conflict
