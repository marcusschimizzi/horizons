  <action>
    **1. Add parallax constants to `src/lib/scene-constants.ts`:**
    - `parallaxMaxX: 0.8` — max X camera shift at full mouse offset (subtle, per CONTEXT.md ~0.5-1 unit)
    - `parallaxMaxY: 0.5` — max Y camera shift (slightly less than X)
    - `parallaxSmoothTime: 0.3` — smoothTime for parallax damping (slightly slower than Z for floaty feel)
    - `snapDistanceThreshold: 1.5` — Z distance from cameraRestZ before snap button appears

    **2. Extend `src/stores/camera-store.ts`:**
    - Add state fields: `targetParallaxX: 0`, `targetParallaxY: 0`
    - Add action `snapToPresent()`: Sets `targetZ = SCENE_CONSTANTS.cameraRestZ`, `isAnimating = true`. Do NOT clear parallax — let it continue naturally.
    - Add action `setParallax(x: number, y: number)`: Sets targetParallaxX = x, targetParallaxY = y, isAnimating = true. These are the TARGET values — damping happens in useFrame.
    - Important: The actual camera X/Y position is managed by damp3 in CameraRig, not stored separately. The store only holds targets.

    **3. Update `src/components/CameraRig.tsx`:**

    a) **Parallax mouse tracking** (new useEffect):
       - Listen for `pointermove` on `gl.domElement`.
       - Normalize mouse position to [-1, 1] range: `nx = ((e.clientX - rect.left) / rect.width) * 2 - 1`, same for ny (inverted: `-ny`).
       - Call `cameraStore.getState().setParallax(nx * parallaxMaxX, -ny * parallaxMaxY)`.
       - On `pointerleave`, gently reset: `cameraStore.getState().setParallax(0, 0)` — lets parallax lerp back to center.
       - Call `invalidate()` on both events.

    b) **Update useFrame damp3 target** from `[0, 0, targetZ]` to `[targetParallaxX, targetParallaxY, targetZ]`.
       - This single damp3 call handles BOTH parallax and Z movement simultaneously.
       - Parallax is depth-stratified automatically: perspective projection means camera X/Y shift causes near objects to move more on screen than far objects. No per-object parallax needed.

    c) **Update settle detection** to include parallax axes:
       - Check all three axes: `Math.abs(camera.position.x - targetParallaxX) < 0.01 && Math.abs(camera.position.y - targetParallaxY) < 0.01 && Math.abs(camera.position.z - targetZ) < 0.01`.
       - Only settle (stop invalidation) when ALL THREE axes are within epsilon.

    d) **Keyboard shortcut** (new useEffect):
       - Listen for `keydown` on `window` (not gl.domElement — keyboard events need window/document).
       - On `Home` key (`e.key === 'Home'`), call `cameraStore.getState().snapToPresent()` and `invalidate()`.
       - Prevent default on Home to avoid browser scroll-to-top.

    **4. Create `src/components/SnapToPresent.tsx`:**

    - Uses `useCameraStore` to reactively subscribe to `currentZ`.
    - Visibility: Only visible when `Math.abs(currentZ - cameraRestZ) > snapDistanceThreshold`. Use CSS opacity + pointer-events (opacity 0 and pointer-events none when hidden, opacity 1 and pointer-events auto when visible) with a CSS transition for smooth fade.
    - Position: Fixed bottom-center of the viewport (e.g., `bottom: 24px, left: 50%, transform: translateX(-50%)`).
    - Design: Minimal dark-space aesthetic. A small pill-shaped button with a subtle border, muted text like "Present" or an upward-arrow icon (use a simple SVG chevron-up or home icon, not an emoji). Background `rgba(10, 10, 15, 0.8)` with `backdrop-filter: blur(8px)`. Border `rgba(148, 163, 184, 0.2)`. Text color `#94a3b8` (matching DebugOverlay secondary). Hover: text brightens to `#e2e8f0`, border brightens.
    - On click: `cameraStore.getState().snapToPresent()`. Also need to call `invalidate` — but this component is outside the Canvas. Solution: the store's `isAnimating = true` will be picked up by CameraRig on the next frame. To trigger that first frame, SnapToPresent should also import `cameraStore` and after calling snapToPresent, the CameraRig's existing SceneInvalidator pattern won't help (it watches task store, not camera store). Instead, add a one-shot invalidation mechanism:
      - Option A (simplest): In CameraRig, add a secondary useEffect that subscribes to `cameraStore` and calls `invalidate()` whenever isAnimating becomes true. This is a tiny subscription that bridges the DOM->Canvas gap.
      - Implement Option A in CameraRig: `useEffect(() => cameraStore.subscribe((state) => { if (state.isAnimating) invalidate(); }), [invalidate])`.
    - Use inline styles (consistent with the project pattern for overlay components like DebugOverlay).

    - Render `<SnapToPresent />` as a sibling to `<Canvas>` and `<DebugOverlay>` in the returned JSX.
