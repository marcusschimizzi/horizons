## v1 Requirements — All Shipped ✓

### Scene

- [x] **SCENE-01**: User sees tasks laid out in a 3D perspective scene with six named horizon bands (Immediate → Someday) mapped to Z-depth positions
- [x] **SCENE-02**: Tasks in distant horizons are visually blurred and shrunk by exponential fog — proximity equals legibility
- [x] **SCENE-03**: Close tasks (Immediate, This Week) render as full Html cards with title, tags, date range, and drift indicator visible
- [x] **SCENE-04**: Distant tasks (This Month and beyond) render as glowing sprites — no text, colored by tag category
- [x] **SCENE-05**: Task nodes transition between LOD tiers (card ↔ sprite) smoothly as camera moves along Z-axis
- [x] **SCENE-06**: Scene atmosphere includes background particles / star field for spatial depth

### Camera

- [x] **CAM-01**: User can scroll to move the camera along the Z-axis (fly into the future / back to the present)
- [x] **CAM-02**: Camera movement has easing / momentum (lerp toward target position, not instant snap)
- [x] **CAM-03**: Camera has soft limits — cannot scroll past Z=0 (present) or beyond Someday depth
- [x] **CAM-04**: User can snap back to the present (Z=0) via button or keyboard shortcut
- [x] **CAM-05**: Subtle mouse-follow parallax adds depth perception as cursor moves across the scene

### Capture

- [x] **CAP-01**: User can type a natural language intention into a fixed input bubble overlaid on the canvas
- [x] **CAP-02**: Input is sent to Haiku via server-side API route and parsed into a structured task (title, targetDate, horizon, tags, needsRefinement)
- [x] **CAP-03**: Parsed task is persisted to Railway Postgres immediately after parsing
- [x] **CAP-04**: New task materializes in the 3D scene at its computed horizon position with an entrance animation (fade-in, scale-up)
- [x] **CAP-05**: Anthropic API key never reaches the client — all AI calls are server-side

### Task Interaction

- [x] **TASK-01**: User can click any task node (card or sprite) to open a TaskDetail panel as a 2D slide-in overlay
- [x] **TASK-02**: User can complete a task from the detail panel — task dissolves and fades from the scene with a satisfying animation
- [x] **TASK-03**: User can drop a task (guilt-free deletion) — distinct from complete, task disappears without ceremony
- [x] **TASK-04**: User can reschedule a task by changing its target date/horizon — task drifts to new Z-position
- [x] **TASK-05**: User can edit a task's title and details inline from the detail panel

### Accountability

- [x] **ACCT-01**: Each task tracks a `driftCount` — increments whenever the task's horizon window passes without completion
- [x] **ACCT-02**: Drift count is visible as a subtle indicator on the task node in the scene
- [x] **ACCT-03**: At drift count 3+, the detail panel surfaces a gentle prompt: "This has been pushed N times — recommit, snooze to Someday, or drop?"
- [x] **ACCT-04**: Tasks flagged `needsRefinement: true` display a distinct visual indicator (pulse, ring) on their scene node
- [x] **ACCT-05**: Clicking a refinement-flagged task shows the AI-generated refinement prompt in the detail panel
- [x] **ACCT-06**: User can respond to refinement prompts in natural language — Haiku updates the task and clears the flag

### Navigation

- [x] **NAV-01**: User can toggle a list view showing all tasks flat-grouped by horizon, as an escape hatch from the 3D scene
- [x] **NAV-02**: List view supports filtering by tag, status, and needs-refinement flag
- [x] **NAV-03**: List view supports quick actions: complete, drop, reschedule without opening the full detail panel

### Visual

- [x] **VIS-01**: Task glow colors are assigned by tag category (e.g. work: cool blue, personal: warm amber, health: green)
- [x] **VIS-02**: Bloom post-processing effect applies to emissive distant task nodes
- [x] **VIS-03**: Tasks with hard deadlines have a visually distinct treatment (sharper, brighter, subtle pulse ring)
